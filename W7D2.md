## 2. Prose is useless to a program (~10 min)

Yesterday Claude replied with a sentence, and you printed it. Fine for a human. But the pipeline isn't a human ‚Äî it needs to *store* the result, query it, put it in a report. A paragraph of prose can't be stored in a column or counted in a query.

So you ask Claude for **structured output**: instead of "write me a paragraph about this brand," you say "return a JSON object with these exact fields." JSON maps directly onto a Python dict (you've used `json` since Week 1), so a JSON reply becomes data your program can use the same way it uses everything else.

The catch: Claude is a text generator. It will happily return JSON *if you ask clearly* ‚Äî but it can also wrap it in a "Here's the analysis:" sentence, or in markdown code fences, if you're sloppy. Today is the happy path: a clear prompt, clean JSON, parse it. D4 deals with what happens when it goes wrong.

## 3. The fields you already have, and the questions they answer (~10 min)

Back in W2D4 you taught `parse_page` to extract five fields. They've been "floating data points" ever since ‚Äî you collected them without a clear use. Today they earn their keep. Each one answers a brand-evaluation question:

| Field | The question it helps answer |
|-------|------------------------------|
| `description` (meta) | What's the brand's own one-line pitch? Is there one at all? |
| `h1` | What's the single biggest message on the page? |
| `title` + `og_title` | Does the brand present itself *consistently* across the tab title, the social-share title, and the headline? |
| `canonical` | Does this US page treat *itself* as the canonical home, or does it point somewhere else (a sign of a thin or non-localized US presence)? |

That last column is the actual product. Brand Lens isn't "a thing that fetches HTML" ‚Äî it's "a thing that turns those fetched fields into a judgment about a brand's US presence." The LLM is what crosses that gap. Today you wire the fields into questions.

## 4. Get the brand's fields out of the database (~15 min)

Add a query to `db.py` that returns the fields you want, as dicts:

```python
def page_fields_for_brand(conn, brand_slug):
    cursor = conn.execute(
        """
        SELECT url, title, description, og_title, h1, canonical
        FROM pages WHERE brand_slug = ?
        """,
        (brand_slug,),
    )
    columns = [col[0] for col in cursor.description]
    return [dict(zip(columns, row)) for row in cursor.fetchall()]
```

One new idea: **`cursor.description`** is metadata the database hands back describing the columns of the result ‚Äî a list where each `col[0]` is a column name. Zipping the names with each row turns a bare tuple `("https://...", "Title", ...)` into a dict `{"url": "https://...", "title": "Title", ...}`. Now the LLM context is built from named fields, not mystery positions.

Check it from the terminal (use a brand you actually have data for):

## 5. Build the prompt and ask for JSON (~20 min)

```python
import json

from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()
client = Anthropic()

SYSTEM = (
    "You are a brand analyst evaluating how a Korean brand presents itself "
    "to the US market, based only on data scraped from its US website. "
    "Respond with ONLY a JSON object ‚Äî no markdown fences, no prose before "
    "or after."
)


def build_prompt(brand_name, pages):
    lines = [f"Brand: {brand_name}", "", "Scraped pages:"]
    for p in pages:
        lines.append(f"- url: {p['url']}")
        lines.append(f"  title: {p['title']}")
        lines.append(f"  meta description: {p['description']}")
        lines.append(f"  og:title: {p['og_title']}")
        lines.append(f"  h1: {p['h1']}")
        lines.append(f"  canonical: {p['canonical']}")
    lines.append("")
    lines.append(
        "Return a JSON object with these keys:\n"
        '  "value_proposition": one sentence, the brand\'s pitch as the data '
        "suggests it (use the meta description and h1).\n"
        '  "messaging_consistency": one sentence on whether title, og:title, '
        "and h1 tell a consistent story.\n"
        '  "us_presence_signal": one sentence on what the canonical URLs '
        "suggest about US-market commitment.\n"
        '  "confidence": one of "low", "medium", "high" ‚Äî how much the scraped '
        "data actually supports your read."
    )
    return "\n".join(lines)


def summarize_brand(brand_name, pages):
    prompt = build_prompt(brand_name, pages)
    message = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=500,
        system=SYSTEM,
        messages=[{"role": "user", "content": prompt}],
    )
    raw = message.content[0].text
    return json.loads(raw)
```

The new pieces:

- **`system=SYSTEM`** ‚Äî a *system prompt*. It sets the role and the rules for the whole call, separate from the user message. Here it does the heavy lifting of "JSON only, no fences" ‚Äî instructions about *how* to answer belong in the system prompt; the actual data goes in the user message.
- **`build_prompt`** is plain Python string-building ‚Äî no LLM involved. It lays out each page's fields and then spells out the exact JSON keys you want. Being explicit about the keys is what makes the output *structured*: you're not hoping Claude picks good keys, you're naming them.
- **`json.loads(raw)`** ‚Äî the same `json` module from Week 1, turning Claude's JSON *text* into a Python dict. After this line, `summarize_brand` returns a dict your program can store and query.

Notice each requested key maps back to the table in section 3: `value_proposition` ‚Üê description/h1, `messaging_consistency` ‚Üê title/og_title/h1, `us_presence_signal` ‚Üê canonical. You're not asking Claude to "analyze the brand" vaguely ‚Äî you're asking four specific questions tied to fields you actually have.

## 7. When it doesn't parse (~5 min)

Eventually you may hit a `json.JSONDecodeError` ‚Äî Claude added a stray sentence, or wrapped the JSON in ```` ```json ```` fences, and `json.loads` choked. **Don't fix it today.** Just notice it: this is the gap between "works when I run it" and "works every time, automatically." A program that calls an LLM has to survive a reply that isn't shaped the way you hoped.
