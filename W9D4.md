## 2. What the build has to do (~10 min)

You predicted this in last night's journal. The job, start to finish:

1. Get the list of brands from the database.
2. For **each** brand: build its context (the stored brief + its pages), render the brief page, write `site/<slug>.html`.
3. Render the index page over all the brands, write `site/index.html`.
4. Copy `static/style.css` into `site/` so the `<link>` resolves.

This is the HTML sibling of Week 8's `generate_briefs.py`, which did steps 1–2 for markdown. You already wrote `build_context` there; you'll reuse it. The new parts are the index (step 3) and the stylesheet copy (step 4).

## 3. Reuse build_context (~10 min)

Open `generate_briefs.py` from Week 8. It has a `build_context(conn, brand)` that pulls the stored brief (with `json.loads`, the round-trip back from W7D4's `json.dumps`) and the brand's pages into the context dict your templates expect. That function is presentation-agnostic — it builds *data*, not markdown — so the HTML build can call it unchanged. Separation of concerns earning its keep (W8D3): the "what data goes in a brief" decision lives in one place, and both the markdown and HTML renderers draw from it.

If `build_context` currently lives inside `generate_briefs.py` and isn't easily importable, move it into `db.py` (it's database-shaped) or leave it and `from generate_briefs import build_context`. Either is fine; the point is one definition, two callers.

## 4. Write build_site.py (~25 min)

```
subl build_site.py
```

```python
"""Rebuild the entire static site from the database into site/."""
import shutil
from datetime import date

import db
from generate_briefs import build_context  # or wherever build_context lives
from render import render_brief_html, render_index, write_page


def build_site():
    conn = db.get_connection()
    today = date.today().isoformat()

    brands = db.all_brands(conn)
    for brand in brands:
        context = build_context(conn, brand["slug"])
        context["generated_at"] = today
        html = render_brief_html(context)
        path = write_page(brand["slug"], html)
        print(f"wrote {path}")

    index_html = render_index(brands, today)
    write_page("index", index_html)
    print("wrote site/index.html")

    shutil.copy("static/style.css", "site/style.css")
    print("copied stylesheet")

    return len(brands)


if __name__ == "__main__":
    count = build_site()
    print(f"built site for {count} brand(s)")
```

Read the new pieces:

- **`shutil.copy("static/style.css", "site/style.css")`** — `shutil` is Python's standard "shell utilities" module for file operations; `copy` does what `cp` did by hand yesterday. Now the build never forgets the stylesheet.
- **`date.today().isoformat()`** — today's date as a string like `"2026-06-18"`, computed once and threaded into every page's `generated_at`. (You hand-typed dates in the throwaway scripts; the real build computes it.)
- **The `__main__` guard** (W3D2) — so `import build_site` in a test won't trigger a full rebuild.

Notice `build_site` *returns* the brand count and prints alongside. The return value is what a test can check; the prints are for the human running it. Two audiences, handled.

## 6. Test the renderers — pure, so no mocking (~20 min)

The renderers are the new code that matters, and they're **pure** (W5D3) — hand them data, get a string, no I/O. So testing them needs no `monkeypatch`, no fakes, none of the machinery from Weeks 5 and 7. This is the W8D4 lesson again: purity is what makes a test a two-liner.

Create `tests/test_build_site.py`:

```python
from render import render_brief_html, render_index


def make_context(brief=None, pages=None):
    return {
        "brand_name": "Innisfree",
        "generated_at": "2026-06-18",
        "brief": brief,
        "pages": pages or [],
    }


def test_brief_html_has_html_skeleton():
    html = render_brief_html(make_context())
    assert "<!DOCTYPE html>" in html
    assert "<h1>" in html


def test_brief_html_shows_brief_when_present():
    brief = {
        "value_proposition": "Clean beauty from Jeju Island",
        "messaging_consistency": "consistent",
        "us_presence_signal": "canonical points to US site",
        "confidence": "medium",
    }
    html = render_brief_html(make_context(brief=brief))
    assert "Clean beauty from Jeju Island" in html


def test_brief_html_fallback_when_no_brief():
    html = render_brief_html(make_context(brief=None))
    assert "No brief" in html


def test_index_links_each_brand():
    brands = [
        {"slug": "innisfree", "name": "Innisfree"},
        {"slug": "laneige", "name": "Laneige"},
    ]
    html = render_index(brands, "2026-06-18")
    assert 'href="innisfree.html"' in html
    assert "Laneige" in html


def test_index_handles_no_brands():
    html = render_index([], "2026-06-18")
    assert "No brands" in html
```

What each one pins:

- The skeleton test guards that you're emitting real HTML, not bare text.
- The present/absent-brief pair pins the `{% if %}/{% else %}` branches (same idea as W8D4's render tests, new template).
- **`test_index_links_each_brand` checks the actual `href`** — `'href="innisfree.html"'` is the literal link the index must build from the slug. If someone breaks the slug-to-filename glue, this fails. That link is load-bearing: get it wrong and every page on the site 404s.
- The empty-brands test pins the `{% for %}/{% else %}` loop-else from D2.

Run:

```

```

All earlier tests plus five new ones, green. Notice again — no mocking, fast to write. That speed is the dividend of keeping the renderers pure.
