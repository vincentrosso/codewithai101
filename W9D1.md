That's the simplest kind of website there is, and (Weeks 11–12) the simplest kind to put on the internet.

## 2. Markdown vs HTML — why the switch (~10 min)

You already know how to render a *document* — that was Week 8. So why a new format?

A **markdown** file (`.md`) is plain text with conventions: `#` means heading, `*` means italic. *You* know how to read it, and tools like GitHub render it prettily, but a web browser pointed at a `.md` file just shows the raw characters. Markdown is for writing; it isn't the web's language.

**HTML** (HyperText Markup Language) *is* the web's language. It's also just text, but its markings are **tags** — `<h1>...</h1>` for a big heading, `<p>...</p>` for a paragraph, `<a href="...">...</a>` for a link. When a browser opens an `.html` file, it reads those tags and *draws* a page: the `<h1>` becomes large bold text, the `<a>` becomes a clickable link. (D3 is all about how that drawing happens.)

The good news: you already know the rendering machinery. `jinja2` doesn't care whether the template is markdown or HTML — it's the same `{{ }}` and `{% %}` you used in Week 8. Only the surrounding text changes, from markdown to HTML tags. Today is "the W8D1 skill, pointed at a new target."

## 3. Write your first HTML template (~25 min)

Your brand brief, as a web page. Create it next to the markdown one:

```
subl templates/brief.html
```

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>{{ brand_name }} — US Presence Brief</title>
</head>
<body>
  <p><a href="index.html">&larr; All brands</a></p>

  <h1>{{ brand_name }} — US Presence Brief</h1>
  <p><em>Generated {{ generated_at }}</em></p>

  <h2>Summary</h2>
  {% if brief %}
  <p><strong>Value proposition:</strong> {{ brief.value_proposition }}</p>
  <p><strong>Messaging consistency:</strong> {{ brief.messaging_consistency }}</p>
  <p><strong>US-presence signal:</strong> {{ brief.us_presence_signal }}</p>
  <p><strong>Confidence:</strong> {{ brief.confidence }}</p>
  {% else %}
  <p><em>No brief has been generated for this brand yet.</em></p>
  {% endif %}

  <h2>Pages analyzed</h2>
  <ul>
    {% for page in pages %}
    <li><a href="{{ page.url }}">{{ page.title or page.url }}</a></li>
    {% endfor %}
  </ul>
</body>
</html>
```

Read it against the markdown version from W8D1 — the *logic* is identical, only the wrapping changed:

- **The `{{ }}` and `{% %}` are exactly the same** as Week 8. `{{ brand_name }}` still drops in the value; `{% if brief %}` still picks a branch; `{% for page in pages %}` still loops. Your template skill transfers untouched.
- **What's new is the HTML scaffolding.** `<!DOCTYPE html>` and `<html>`, `<head>`, `<body>` are the skeleton every web page has. The `<head>` holds page *metadata* (the `<title>` is what shows in the browser tab); the `<body>` holds what you *see*.
- **The tags map onto the markdown you knew.** `# Heading` → `<h1>Heading</h1>`. `**bold**` → `<strong>bold</strong>`. A markdown link `[text](url)` → `<a href="url">text</a>`. A bullet list becomes `<ul>` (the list) wrapping `<li>` items (each bullet).
- **Notice the `<a href="index.html">` at the top.** That's a link to the index page you'll build tomorrow — a *relative* link, meaning "the file `index.html` next to me." Relative links are how the pages of a static site find each other (much more on this D3).

## 4. Render HTML from Python — one new wrinkle (~20 min)

Open `render.py`. Right now its `Environment` looks like this from Week 8:

```python
env = Environment(loader=FileSystemLoader("templates"))
```

For HTML, change that one line to turn on **autoescaping**:

```python
from jinja2 import Environment, FileSystemLoader, select_autoescape

env = Environment(
    loader=FileSystemLoader("templates"),
    autoescape=select_autoescape(["html"]),
)
```

Here's what that does and why it matters. Your data goes *into* HTML now, and HTML has special characters — `<`, `>`, `&`. Imagine a brand whose value proposition contained the literal text `value < price`. Dropped raw into the page, the browser would see `<` and start hunting for a tag that isn't there, mangling the page. **Autoescaping** makes `jinja2` automatically convert those special characters into safe equivalents (`<` becomes `&lt;`) so your *data* always renders as text, never as accidental HTML. `select_autoescape(["html"])` says "escape for templates ending in `.html`, leave `.md` alone." It's a small line with a real safety job — you'll meet the grown-up version of this idea (don't let data be mistaken for code) again and again.

Now add a function that renders the HTML brief. Your existing `render_brief` loads `brief.md`; add a sibling for HTML:

```python
def render_brief_html(context):
    """Turn a context dict into the rendered HTML string. Pure — no I/O."""
    template = env.get_template("brief.html")
    return template.render(**context)
```

Same shape as `render_brief`, same purity — hand it a context, get a string, no file touched. (That purity is, again, why D4's tests will be trivial.)

Try it with the same hand-made data trick from W8D1 (throwaway script):

```

```

## 5. Write it to a file and open it (~15 min)

The site lives in its own directory, `site/` — the HTML equivalent of Week 8's `briefs/`. Add a writer to `render.py`:

```python
import os


def write_page(slug, html):
    os.makedirs("site", exist_ok=True)
    path = os.path.join("site", f"{slug}.html")
    with open(path, "w", encoding="utf-8") as f:
        f.write(html)
    return path
```

Same pattern as `write_brief` from W8D1 — pure renderer, separate side-effectful writer. Update the bottom of `try_html.py` to write instead of print:

```python
html = render_brief_html(context)
path = write_page("innisfree", html)
print(f"wrote {path}")
```

```
python3 try_html.py
```

Now open it **in your browser** — on a Mac, `open` hands a file to your default app:

```
open site/innisfree.html
```

A real web page. Headings rendered, the link clickable (the "All brands" link is dead until tomorrow — that's expected). It's unstyled and a little plain; that's tomorrow's stylesheet. But it's *the web now*, not a text file.

**Keep the generated site out of git.** Like `briefs/` and `brand_lens.db`, it's regenerable output. Add to `.gitignore`:

```

```
