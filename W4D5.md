# Week 4, Day 5 — Questions You Can't Answer Yet
*v1.0.0*

**Goal for today:** no Vincent session this week. Sit down with what you've learned and write a list of specific questions you can't answer right now without help. The list itself is the deliverable. Future weeks may answer some; the rest become Vincent fodder.

**Time budget:** ~60 min question writing, ~30 min retrospective journal, ~30 min slack.

---

## 1. Why this day exists (~5 min)

The first three weeks ended with a mentor review where Vincent asked you to explain your code from memory. Week 4 ends differently. The question this week isn't "can you explain what you wrote?" — it's "what don't you understand yet about working with Claude Code?"

Naming gaps is harder than answering questions. You have to notice the edge of your knowledge and put it into words. That's the work today.

## 2. The exercise — ground rules (~5 min)

You're going to write a list of at least **10 specific questions** you couldn't answer right now without looking something up or asking someone. Two ground rules:

- **Specific, not vague.** Not "I don't get diffs." Yes "I don't understand why `git diff` sometimes shows whitespace changes I didn't make." The first is a topic; the second is a question.
- **Honest.** Don't write questions you already know the answer to in order to look diligent. Write the ones that actually nag at you.

If you can't get to 10, look back at this week's journals. The "what I don't fully understand" sections are full of seed material.

## 3. Brainstorm (~30 min)

Open `journal/W4D5_questions.md`. Write your list. Useful categories to draw from — but feel free to ignore them if your real questions don't fit:

- **About Claude Code itself.** When does it ask permission and when doesn't it? Where does it store conversation history? What can it not do?
- **About your own code.** A line in `runner.py`, `fetch.py`, or `parse.py` that, after this week, you realize you still can't explain.
- **About diffs and git.** Something from W4D3 that didn't fully click.
- **About the Week 4 workflow.** Where did the "review every diff" rule feel like overkill? Where did it feel insufficient?
- **About things Claude Code got wrong.** A claim Claude Code made this week that you suspect was wrong but couldn't verify.

Quality bar: each question should be **answerable**. "Why is programming hard" is not. "Why does my venv not auto-activate when I `cd` into the repo" is.

## 4. Categorize each (~15 min)

Below the list, go through each question and tag it with one of:

- **`(claude code)`** — you'd ask Claude Code itself.
- **`(docs)`** — you'd find this in official documentation: Python docs, git docs, Anthropic docs.
- **`(vincent)`** — you'd save this for the next mentor session.
- **`(later)`** — the question is real but the answer can wait — you don't have the prerequisite knowledge to even understand the answer yet.

The categorization matters because it teaches you something about your own questions. Most learners over-tag `(vincent)` and under-tag `(docs)`. Push back on yourself.

## 5. Save the list and commit (~5 min)

```
git add journal/W4D5_questions.md
git commit -m "week 4: questions i can't answer yet"
git push
```

This list is reference material. Future weeks will revisit it. Don't delete it when the week ends.

## 6. Week 4 retrospective journal (~30 min)

In a separate file, `journal/W4D5.md`:

```
# Week 4 retrospective

## What I shipped this week
Concretely: what's different in the brand-lens repo as of Friday vs Monday morning?

## How my relationship to Claude changed
You met Claude in a browser tab on W2D1. You met Claude Code on W4D1. They're the same model. What's actually different about working with each, in your experience?

## A specific moment I broke a rule
The Week 4 rules: review every diff, never auto-approve, ask before committing unclear code. Did you break one? Which, when, what happened?

## Where the diff review reflex felt strongest
A specific diff this week where you caught something by reading it carefully — something you would have missed if you'd just hit accept.

## Where the diff review reflex felt weakest
A specific diff this week where you accepted something faster than you should have. (If you can't think of one — that itself is suspicious. Be honest.)

## My top 3 questions from W4D5_questions.md
The three you'd most want to answer first.
```
